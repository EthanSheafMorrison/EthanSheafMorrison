Dynamic Project System Instructions

I've set up a dynamic project system that allows you to create project pages using a single HTML template and markdown files. Here's how it works and how to view the example.

Viewing the Example

I've created an example using your "Finding Kuku" project from data.json. To view it:

View Finding Kuku Example

You can also view the list of all projects on the index page:

View All Projects

How It Works

  1. Single HTML Template: The projects.html file serves as a template for all project pages
  2. Dynamic Loading: It loads data from your existing data.json file in the root directory
  3. Project Content: Detailed content is stored in markdown files in the projects/data/ directory
  4. URL Parameters: The project to display is determined by the ID in the URL (e.g., ?id=finding-kuku)

Directory Structure

projects/
├── data/
│   ├── finding-kuku.md        # Markdown content for Finding Kuku project
│   └── sample-project.md      # Sample markdown template
├── index.html                 # List of all projects
├── projects.html              # Template for displaying individual projects
└── viewing-instructions.html  # This file

images/projects/
└── finding-kuku/             # Images for the Finding Kuku project
    ├── main.jpg              # Main project image
    ├── thumbnail.jpg         # Thumbnail for project cards
    ├── image1.jpg            # Additional project images
    └── image2.jpg            # Additional project images

Adding a New Project

To add a new project:

  1. Add the project info to your data.json file in the root directory
  2. Create a markdown file in projects/data/ named after the project ID (e.g., project-id.md)
  3. Add project images to images/projects/project-id/ directory

Markdown Format

Each markdown file should have front matter at the top and content sections below:

---
title: Project Title
date: 2024-01-15
image_caption: A brief caption for the main image
---

## Section Title

Content paragraphs here...

## Another Section

- List item
- Another list item

Image Naming Convention

For the project images to load correctly, use this naming convention:

  • main.jpg - The hero image that appears at the top of the project page
  • thumbnail.jpg - The thumbnail image used on the projects index page
  • image1.jpg, image2.jpg, image3.jpg - Additional project images

Note: If an image doesn't exist, the system will handle it gracefully by either removing the image placeholder or using a default placeholder image.
